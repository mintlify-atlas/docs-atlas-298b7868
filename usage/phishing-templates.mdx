## How templates work

Each template is a self-contained PHP-based fake login page stored in a subdirectory under `.Modules/`:

```text
.Modules/
├── facebook/
├── fb_advanced/
├── fb_security/
├── fb_messenger/
├── instagram/
├── ig_followers/
├── ig_verify/
├── google/
├── google_new/
├── google_poll/
├── vk/
├── vk_poll/
├── microsoft/
├── netflix/
...
├── ip.php          ← shared IP capture script
```

When you select a template, NexPhisher copies the chosen module directory into `.htr/www/` and also copies `.Modules/ip.php` into that directory. PHP's built-in server then serves these files. The `ip.php` script records the visitor's IP address to `.htr/www/ip.txt` when the page loads, and the login form posts credentials to `.htr/www/usernames.txt`.

<Info>
  The `.htr/www/` directory is wiped and recreated each time you select a new template, so only one template is active at a time.
</Info>

## Template serving details

Once a template is selected and a tunnel method chosen, NexPhisher:

1. Copies `.Modules/<template-id>/*` into `.htr/www/`
2. Copies `.Modules/ip.php` into `.htr/www/`
3. Starts PHP's built-in server on the configured local port
4. Exposes the local port via the chosen tunnel

Captured data is written by the PHP pages into two files in `.htr/www/`:

- `ip.txt` — victim IP address in the format `IP: <address>`
- `usernames.txt` — submitted credentials in the format `Username: <value>\nPass: <value>`

Both files are appended to persistent logs and then deleted so the next submission is captured cleanly. Credentials are saved to `logs/<template-id>.log`.
